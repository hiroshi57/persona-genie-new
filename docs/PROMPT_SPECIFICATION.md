# Persona Genie Pro - プロンプト仕様書

## 📋 概要

このドキュメントは、Google Gemini APIに送信するプロンプトの仕様を詳細に説明しています。

---

## 🎯 プロンプト目的

商品情報から3パターンの顧客ペルソナを生成し、各ペルソナについて以下を提供：

1. ✅ 基本情報（人口統計学的属性）
2. ✅ AIDMA分析（購買心理プロセス）
3. ✅ カスタマージャーニー（4段階の顧客体験）
4. ✅ PDCA施策（マーケティング戦略）
5. ✅ 心理転換点（購入決定トリガー）
6. ✅ データソース（100%検証済みの参考資料）

---

## 🔒 絶対的な制約条件

### 信頼性スコア
```
✅ 常に100に設定
❌ 100未満の情報は除外
❌ 曖昧な予測は含めない
```

### データソース
```
✅ 必ずURL付きで記述
✅ 公式統計、政府機関データを優先
✅ 学術調査・研究レポートを参照
❌ 個人ブログやWikipedia は参考にしない
❌ URLなしのデータは記述しない
```

### ペルソナの実現性
```
✅ 統計データに基づいた属性のみ
✅ 日本の人口動態に基づく
✅ 実在する可能性が高いペルソナ
❌ 架空の属性の組み合わせ
❌ 完全に想像上のキャラクター
```

## 📊 3パターンのペルソナ構造

### ペルソナA: 想定通り × 期待通り

**定義**:
- 顧客仮説通りのターゲット
- 想定通りの理由で購入

**例**:
```
商品: スマートウォッチ
ターゲット仮説: 30代営業職
期待される価値: 健康管理とスケジュール管理
ペルソナA: 35歳男性営業職が、毎日の健康管理とビジネス効率化目的で購入
```

### ペルソナB: 想定外 × 期待通り

**定義**:
- 予想外の顧客層が
- 期待通りの価値を求める

**例**:
```
商品: スマートウォッチ
想定外ターゲット: 60代女性リタイア層
期待される価値: 健康管理機能
ペルソナB: 63歳女性リタイア層が、健康寿命延伸目的で購入
```

### ペルソナC: 想定通り × 想定外

**定義**:
- 想定通りのターゲットが
- 想定外の価値を見出す

**例**:
```
商品: スマートウォッチ
ターゲット仮説: 30代営業職
想定外の価値: スポーツ・フィットネス活動追跡
ペルソナC: 35歳男性営業職がアスリート志向で、トレーニングデータ記録目的で購入
```

## 📐 各セクションの詳細仕様

### 1. 基本情報セクション

**必須フィールド**:
```json
{
  "基本情報": {
    "名前": "具体的な日本人名",
    "年齢": "数値",
    "性別": "性別",
    "職業": "職業（国勢調査に基づく）",
    "経済状況": "年収レベル（総務省統計ベース）",
    "教育レベル": "学歴",
    "家族構成": "家族構成",
    "居住地": "都道府県"
  }
}
```

**仕様**:
- **名前**: 実在する日本人名（性別に応じた名前）
  - ✅ 田中太郎、山田花子
  - ❌ AI-1号、ペルソナA
- **年齢**: 必ず具体的な数値
  - ✅ 35歳、62歳
  - ❌ 30代、中年
- **職業**: 日本の職業統計に存在する職業
  - 参考: 厚生労働省「職業分類」
- **経済状況**: 総務省「家計調査」の年収階級に基づく
  - 例: 年収500～700万円、年収1000万円以上
- **居住地**: 具体的な都道府県（地域でもOK）
  - ✅ 東京都渋谷区、大阪府大阪市
  - ❌ 関東地域

### 2. AIDMA分析セクション

**5段階の詳細仕様**:

```
認知 (Awareness)
↓
興味 (Interest)
↓
欲求 (Desire)
↓
記憶 (Memory)
↓
行動 (Action)
```

**各段階のフォーマット**:

| 段階 | 説明 | 例 |
|------|------|-----|
| 認知 | どのメディア/場所で製品を知ったか | 「Googleの検索結果で見かけた」「駅の広告で見た」 |
| 興味 | 何に惹かれたのか | 「デザインの良さ」「機能の豊富さ」 |
| 欲求 | なぜそれが必要か | 「仕事の効率を上げたい」「健康を管理したい」 |
| 記憶 | どのようにリサーチしたか | 「Amazonのレビューを読んだ」「友人に相談した」 |
| 行動 | 購入に至った決定的な理由 | 「限定セールで割引があったから」「返品保証があったから」 |

**重要**: 各段階は「具体的なタッチポイント」を含める

### 3. カスタマージャーニーセクション

**4段階の構造**:

```
認知段階 → 検討段階 → 決定段階 → 購入段階
```

**各段階の必須要素**:

```json
{
  "段階": "段階の名前",
  "状況": "ペルソナがどのような状況にあるか",
  "心理状態": "そのときの心理（感情）",
  "行動": "具体的な行動",
  "タッチポイント": "接触するメディア・場所"
}
```

**心理状態の例**:
- 認知段階: 「問題に気づき、不安を感じている」
- 検討段階: 「複数の選択肢に迷っている」
- 決定段階: 「購入への確信が高まり、期待感が増している」
- 購入段階: 「決断の確信と期待感のピーク」

**タッチポイントの例**:
- Google検索、YouTube動画、SNS（Twitter/Instagram）
- 店舗、ECサイト、友人の推薦
- 新聞広告、雑誌、テレビCM

### 4. PDCA施策セクション

**4段階のマーケティング施策**:

| 段階 | 内容 | 具体例 |
|------|------|-------|
| **Plan** | どのような施策を計画するか | 「30代営業職向けのInstagram広告キャンペーンを展開」 |
| **Do** | 実際に実行する施策 | 「朝7時のポドキャスト枠で30秒広告を配信」 |
| **Check** | 効果測定方法と指標 | 「クリック率（CTR）10%、コンバージョン率（CVR）5%」 |
| **Action** | 改善内容 | 「効果が低かった場合、広告タイミングを夜間に変更」 |

**重要**: 各段階は「このペルソナに最適な施策」を記述

### 5. 心理転換点セクション

**構造**: 最低3つ以上のトリガーを時系列で記述

```json
[
  {
    "トリガー": "具体的なきっかけ",
    "発生場面": "いつ、どこで、何があったか",
    "心理的変化": "心理の流れ（例：不安→信頼→購買欲）"
  },
  // 第2、第3のトリガー
]
```

**心理的変化の流れの例**:
- 「問題認識 → 情報探索 → 比較検討 → 確信 → 購買欲」
- 「疑い → 口コミで信頼 → 必要性認識 → 購買意欲」
- 「無関心 → SNS投稿で認識 → 友人推薦で信頼 → 購買決定」

### 6. データソースセクション

**最も重要なセクション - 100%検証済みデータのみ**

```json
{
  "データソース": {
    "1次データソース": "URL付きの公式データ",
    "2次データソース": "URL付きの学術調査",
    "信頼性スコア": 100,
    "データ根拠": "このペルソナが検証済みである理由"
  }
}
```

**1次データソース（政府統計）の優先順位**:

1. **総務省統計局** https://www.stat.go.jp/
   - 国勢調査（人口、世帯、産業分布）
   - 労働力調査（就業状況）
   - 家計調査（収入、支出）

2. **厚生労働省** https://www.mhlw.go.jp/
   - 職業分類
   - 賃金構造基本統計
   - 雇用保険統計

3. **国税庁** https://www.nta.go.jp/
   - 民間給与実態統計
   - 年収データ

4. **経済産業省** https://www.meti.go.jp/
   - 商業統計
   - 消費者動向調査

**2次データソース（学術調査・研究機関）**:

1. **日本マーケティング協会** https://www.jma2.or.jp/
2. **NHK放送文化研究所** https://www.nhk.or.jp/bunken/
3. **電通** https://www.dentsu.co.jp/
4. **博報堂** https://www.hakuhodo.co.jp/
5. **野村総合研究所** https://www.nri.com/jp

**URL付きの記述例**:

```
❌ 悪い例
「日本人の生活時間は平均7時間です」

✅ 良い例
「総務省『社会生活基本調査 https://www.stat.go.jp/』によると、30代の平均睡眠時間は7時間23分です」

✅ より詳細な例
「NHK放送文化研究所『日本人の生活時間調査 https://www.nhk.or.jp/bunken/』と国勢調査データ https://www.stat.go.jp/ に基づくと、35歳会社員の通勤時間は平均1時間15分です」
```

### 定期的な確認項目

1. **データソースの有効性**
   - URLが生きているか確認
   - 統計が最新か確認
   - より信頼度の高いソースがあるか確認

2. **ペルソナの現実性**
   - 統計データと矛盾していないか
   - 日本の市場動向と合致しているか
   - 実際の顧客データと比較

3. **プロンプト文言**
   - より明確な指示になっているか
   - 曖昧な表現がないか
   - 制約条件が十分か

## 📞 よくある質問

### Q: データソースが見つからない場合は？
**A**: 別の公式統計を探すか、その属性を調整してください。架空のURLは絶対に作成しないでください。

### Q: 信頼性スコアを100未満にすることはできますか？
**A**: いいえ、プロンプトで「常に100」と指示されているため、100未満になったら修正が必要です。

### Q: 複数のデータソースを組み合わせてもいいですか？
**A**: はい、むしろ推奨されます。複数の信頼できるソースの交差検証が理想的です。

### Q: ペルソナが完全に同じ属性になった場合は？
**A**: パターンA・B・Cが異なるストーリーで購入理由が異なっていれば、基本属性が同じでも問題ありません。
