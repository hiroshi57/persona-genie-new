## 📊 問題と解決策

### ❌ 旧版の問題
```
入力項目が少ない
├─ 商品名
├─ カテゴリー
└─ [シミュレーションボタン]

結果：
❌ 統計的に有効なシミュレーションができない
❌ 年齢分布の情報がない
❌ 所得層の情報がない
❌ 販売チャネルの情報が不足

```

```
✅ 1. 商品名 ★
✅ 2. カテゴリー ★（ドロップダウン）
✅ 3. 価格帯 ★（¥1K～5K / ¥5K～50K / ¥50K～）

【第2セクション：ターゲット層】
✅ 4. ターゲット年齢層 ★（スライダー：20～70歳）
✅ 5. ターゲット年収層 ★（低/中/高/超高/全階層）
✅ 6. 販売チャネル ★（オンライン/実店舗/両方）

【第3セクション：製品・価値】
✅ 7. 製品の主な特徴 ★（テキスト）
✅ 8. 提供する価値・ソリューション（オプション）

結果：
✅ 1000人の統計的に有効なシミュレーション
✅ 年齢分布（正規分布で生成）
✅ 所得層分布
✅ 認知度スコア（0-100）
✅ 購買意欲スコア（0-100）

```

## 📊 計算ロジック

### 1000人の年齢分布

```javascript
// Box-Muller変換で正規分布を生成
targetAge = 40 + randomNormal() * 15

例：targetAgeRange = [25, 45]
→ 平均35歳、標準偏差15歳の正規分布
→ 1000人のランダム年齢生成
```

### 認知度スコア（0～100）

```javascript
awarenessScore = 
  baseAwareness[販売チャネル]      // 25～50点
  + (priceAdjustment × 0.2)        // 価格調整の影響
  + (salesExpansion × 0.3)         // チャネル拡大の影響
  + (regionDiversity × 0.2)        // 地域多様性の影響
  + randomNormal()                 // ランダムノイズ

範囲調整：min(0, max(100, score))
```

### 購買意欲スコア（0～100）

```javascript
purchaseScore = 
  basePurchase[価格帯]             // 20～60点
  + (incomeLevel × 0.3)            // 所得水準の影響
  + (ageMatch × 0.2)               // 年齢マッチの影響
  + (priceAdjustment × 0.1)        // 価格調整の影響
  + randomNormal()                 // ランダムノイズ

範囲調整：min(0, max(100, score))
```

### 4象限判定

```javascript
if (awareness >= 50 && purchase >= 50) {
  quadrant = "Q1"  // 高認知×高欲望 → 優先度：最高 🔴
} else if (awareness < 50 && purchase >= 50) {
  quadrant = "Q2"  // 低認知×高欲望 → マーケ必須 🟡
} else if (awareness < 50 && purchase < 50) {
  quadrant = "Q3"  // 低認知×低欲望 → 開拓層 🔵
} else {
  quadrant = "Q4"  // 高認知×低欲望 → 見直し層 🟢
}
```

## 🎯 ユースケース

### ケース1：新製品発売のターゲット分析

```
商品：新作ゲーミングノートPC
価格帯：¥50K～
ターゲット：20～35歳の中～高所得男性
販売：オンライン＋実店舗

結果：
Q1（高認知×高欲望）：35% → 優先層（獲得施策集中）
Q2（低認知×高欲望）：28% → マーケ強化層（広告投資）
Q3（低認知×低欲望）：22% → 開拓層（教育コンテンツ）
Q4（高認知×低欲望）：15% → 見直し層（価値再提案）
```

```
商品：スマートスピーカー
価格帯：¥1K～5K
ターゲット：全年齢層（シニア層拡大狙い）

結果：
Q1の拡大を目指して、地域多様性スライダーを上げる
→ 地方シニア層の認知度がアップ
→ 販売チャネルを「両方」に変更してテスト
```
