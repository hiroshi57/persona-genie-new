# 📊 Persona Genie Pro v2 - 改良版セットアップガイド

## 問題点の修正内容

### ❌ 旧版の問題
- 入力欄が「商品名」と「カテゴリー」のみ
- 1000人シミュレーションに必要な統計情報が不足
- 4象限分析が精度不足

### ✅ 改良版の改善
- **8つの必須入力項目**を追加
- 年齢、所得、販売チャネルなどの情報が確保
- 統計的に有効な1000人シミュレーションが可能
- 正確な4象限分析が実現可能

---

## 📋 入力項目の詳細

### 第1セクション：商品基本（3項目）

#### 1️⃣ 商品名 ★必須
- **入力欄**：テキスト入力
- **例**：スマートプロジェクター
- **用途**：UI表示、レポート生成

#### 2️⃣ カテゴリー ★必須
- **入力欄**：ドロップダウン選択
- **選択肢**：
  - 電子機器
  - 生活家電
  - スマートホーム
  - 美容健康
  - 食品飲料
  - ファッション
  - その他
- **用途**：シミュレーション人口の業界別特性を決定

#### 3️⃣ 価格帯 ★必須
- **入力欄**：ボタン選択
- **選択肢**：
  - ¥1K～5K → 3,000円
  - ¥5K～50K → 25,000円
  - ¥50K～ → 100,000円
- **用途**：購買意欲スコア計算に直結

---

### 第2セクション：ターゲット層（3項目）

#### 4️⃣ ターゲット年齢層 ★必須
- **入力欄**：スライダー範囲選択（20～70歳）
- **例**：25～45歳
- **用途**：
  - 1000人の年齢分布を決定（Box-Muller正規分布）
  - 認知度スコアに影響
  - 購買意欲スコアに影響

#### 5️⃣ ターゲット年収層 ★必須
- **入力欄**：ドロップダウン選択
- **選択肢**：
  - 低所得層（年収200～400万円）
  - 中所得層（年収400～700万円）
  - 高所得層（年収700～1000万円）
  - 超高所得層（年収1000万円以上）
  - 幅広い層（全所得層対象）
- **用途**：incomeTargetLevel パラメータを決定

#### 6️⃣ 販売チャネル ★必須
- **入力欄**：ボタン選択
- **選択肢**：
  - オンライン → salesLocationExpansion: 30%
  - 実店舗 → salesLocationExpansion: 40%
  - 両方 → salesLocationExpansion: 65%
- **用途**：認知度スコアの初期値を決定

---

### 第3セクション：製品・価値（2項目）

#### 7️⃣ 製品の主な特徴 ★必須
- **入力欄**：テキストエリア（複数行）
- **例**：4K映像対応、自動焦点調整、スマートホーム連携、バッテリー8時間、軽量設計
- **用途**：
  - 認知度スコアの修正係数として使用
  - ペルソナ生成時の説明資料として使用
  - AI分析の基礎情報

#### 8️⃣ 提供する価値・ソリューション（オプション）
- **入力欄**：テキストエリア（複数行）
- **例**：時間短縮、生活の質向上、エンターテイメント充実、ストレス軽減
- **用途**：
  - 購買意欲スコアの修正係数として使用
  - ペルソナの購入理由として使用
  - 象限判定の補助情報

## 📊 シミュレーションで計算される値

### 認知度スコア（awarenessScore）

```javascript
基本値：
- オンライン販売 → 35点
- 実店舗販売 → 25点
- 両方 → 50点

修正係数：
+ 販売チャネル拡大スライダー（-50～+50）
+ 地域多様性（0～100）
+ 製品特徴の情報量（0～20点）

結果：0～100の正規分布
```

### 購買意欲スコア（purchaseIntentScore）

```javascript
基本値：
- 価格帯が低いほど高い → 60点
- 中価格帯 → 40点
- 高価格帯 → 20点

修正係数：
+ 所得水準（-30～+30）
+ 価格調整スライダー（-50～+50）
+ ターゲット年齢マッチ（-20～+20）
+ 提供価値の度合い（0～20点）

結果：0～100の正規分布
```

### 象限判定

```javascript
Q1（高認知×高欲望）：awareness ≥ 50 && desire ≥ 50
  → 優先度：最高 🔴

Q2（低認知×高欲望）：awareness < 50 && desire ≥ 50
  → マーケ必須 🟡

Q3（低認知×低欲望）：awareness < 50 && desire < 50
  → 開拓層 🔵

Q4（高認知×低欲望）：awareness ≥ 50 && desire < 50
  → 見直し層 🟢
```

## 🔧 カスタマイズ例

### カテゴリーを追加したい場合

**App.tsx内のカテゴリー選択**を修正：

```jsx
<option value="新カテゴリー名">新カテゴリー名（説明）</option>
```

### 価格帯を変更したい場合

**App.tsx内の価格ボタン**を修正：

```jsx
{
  { label: "カスタム価格1", value: 15000 },
  { label: "カスタム価格2", value: 75000 },
  { label: "カスタム価格3", value: 200000 },
}
```

### 年齢範囲を変更したい場合

**スライダーのmin/max属性**を修正：

```jsx
<input type="range" min="16" max="80" />
```
